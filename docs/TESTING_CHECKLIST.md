# PandaDuck Fix 테스트 체크리스트

## 1. 접근성 테스트 (Accessibility Testing)

### 1.1 키보드 네비게이션
- [ ] **Tab 키로 모든 인터랙티브 요소에 접근 가능**
  - [ ] 모든 버튼, 링크, 입력 필드에 Tab으로 접근 가능
  - [ ] 포커스 순서가 논리적임 (왼쪽→오른쪽, 위→아래)
  - [ ] 포커스가 명확하게 표시됨 (파란색 아웃라인)

- [ ] **Enter/Space 키로 버튼 활성화**
  - [ ] 모든 버튼을 Enter 또는 Space로 클릭 가능
  - [ ] 링크는 Enter 키로 작동

- [ ] **Escape 키로 모달/드로어 닫기**
  - [ ] 메뉴 드로어가 Escape로 닫힘
  - [ ] 모달이 Escape로 닫힘

### 1.2 스크린 리더 테스트
**도구:** VoiceOver (macOS/iOS) 또는 NVDA (Windows)

- [ ] **이미지 대체 텍스트**
  - [ ] 모든 이미지에 `alt` 속성 있음
  - [ ] 의미 있는 이미지는 설명적인 alt 텍스트
  - [ ] 장식용 이미지는 `alt=""` 또는 `aria-hidden="true"`

- [ ] **ARIA 레이블**
  - [ ] 아이콘 전용 버튼에 `aria-label` 있음
  - [ ] 링크와 버튼이 명확하게 식별됨
  - [ ] 폼 입력 필드에 라벨 연결됨

- [ ] **콘텐츠 순서**
  - [ ] 스크린 리더가 콘텐츠를 올바른 순서로 읽음
  - [ ] 건너뛰기 링크 (Skip to content) 작동

### 1.3 색상 대비율
**도구:** [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/)

- [ ] **텍스트 대비**
  - [ ] 일반 텍스트: 최소 4.5:1
  - [ ] 큰 텍스트 (18pt+): 최소 3:1
  - [ ] #1D1D1F (본문) vs #FFFFFF 배경: ✅ 15.3:1
  - [ ] #86868B (보조 텍스트) vs #FFFFFF 배경: ✅ 4.6:1

- [ ] **인터랙티브 요소**
  - [ ] 링크와 버튼 색상이 충분한 대비를 가짐
  - [ ] 포커스 상태가 명확히 구별됨

### 1.4 모션 감소
- [ ] **prefers-reduced-motion 지원**
  - [ ] 시스템 설정에서 모션 감소 시 애니메이션 비활성화
  - [ ] `accessibility.css`에 미디어 쿼리 구현됨

### 1.5 폼 접근성
- [ ] **입력 필드**
  - [ ] 모든 입력 필드에 라벨 있음
  - [ ] 필수 필드가 명시됨
  - [ ] 에러 메시지가 접근 가능함

- [ ] **유효성 검사**
  - [ ] 실시간 피드백 제공
  - [ ] 에러가 명확하게 설명됨
  - [ ] `aria-invalid` 속성 사용

## 2. 모바일 테스트 (Mobile Testing)

### 2.1 반응형 디자인
**테스트 기기:**
- iPhone SE (375x667) - 작은 화면
- iPhone 12/13/14 (390x844) - 표준
- iPhone 14 Pro Max (430x932) - 대형
- Android (360x800) - 안드로이드 표준

- [ ] **레이아웃**
  - [ ] 모든 화면 크기에서 컨텐츠가 잘리지 않음
  - [ ] 텍스트가 읽기 쉬운 크기임 (최소 16px)
  - [ ] 버튼이 충분히 큼 (최소 44x44px)

- [ ] **터치 타겟**
  - [ ] 모든 터치 요소가 최소 44x44px
  - [ ] 버튼 간 간격이 충분함 (최소 8px)

### 2.2 iOS 테스트
- [ ] **Safari (iOS)**
  - [ ] 주소창 자동 확장/축소 처리
  - [ ] 100vh 대신 100dvh 사용 (동적 뷰포트)
  - [ - 스크롤이 부드러움 (-webkit-overflow-scrolling: touch)
  - [ ] input 포커스 시 줌 방지 (font-size 16px+)

- [ ] **PWA (홈 화면에 추가)**
  - [ ] 앱이 독립적으로 실행됨 (standalone 모드)
  - [ ] 상태 바 색상이 올바름
  - [ ] 스플래시 화면 표시

### 2.3 안드로이드 테스트
- [ ] **Chrome**
  - [ ] 레이아웃이 정상적임
  - [ ] 터치 스크롤이 부드러움
  - [ ] 백 버튼 작동

- [ ] **PWA**
  - [ ] 설치 프롬프트 표시
  - [ ] 홈 화면에서 아이콘 표시
  - [ ] 스플래시 화면 표시

### 2.3 터치 제스처
- [ ] **스와이프**
  - [ ] 좌/우 스와이프가 직관적임
  - [ ] Pull-to-refresh가 작동 (구현된 경우)

- [ ] **핀치 줌**
  - [ ] 이미지가 줌 가능
  - [ ] 텍스트가 줌 가능 (접근성)

### 2.4 성능
- [ ] **로딩 속도**
  - [ ] 초기 로딩이 3초 이내
  - [ ] Skeleton UI가 로딩 중 표시
  - [ ] 이미지가 lazy loading됨

- [ ] **인터랙션**
  - [ ] 버튼 클릭 후 즉시 피드백
  - [ ] 페이지 전환이 부드러움
  - [ ] 스크롤이 끊기지 않음 (60fps)

## 테스트 수행 방법

### 자동화된 도구
1. **Lighthouse** (Chrome DevTools)
   - Open DevTools → Lighthouse
   - 모든 카테고리에서 90점 이상 목표

2. **axe DevTools**
   - 접근성 이슈 자동 감지
   - 모든 Critical 이슈 해결

3. **WAVE** (웹 접근성 평가 도구)
   - https://wave.webaim.org/
   - URL 입력하여 접근성 검사

### 수동 테스트
1. **키보드만 사용하여 사이트 탐색**
2. **VoiceOver/NVDA로 스크린 리더 테스트**
3. **실제 기기에서 모바일 테스트**
4. **네트워크 스로틀링으로 느린 연결 테스트**
